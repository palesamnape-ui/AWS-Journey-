## Helpful Hints

- Ensure all your resources are created in the same AWS Region for consistency.
- Youâ€™ll need an AWS Identity and Access Management (IAM) role for your Lambda function to access other AWS services. Since we canâ€™t create a new IAM role in this lab, use the `LambdaAccessRole`. This role provides the following permissions:
  - **AWSLambdaBasicExecutionRole**: Grants write permissions to Amazon CloudWatch Logs.
  - **AmazonSNSFullAccess**: Gives full access to Amazon SNS via the AWS Management Console.
  - **AmazonS3FullAccess**: Allows full access to all S3 buckets.
  - **CloudWatchFullAccess**: Provides full access to Amazon CloudWatch.
